# Viamaps テスト仕様書

Google マイマップ × tabiori の機能を持つ旅行地図サービス

2026年5月 / Version 2.0

---

## 1. テスト方針

| テスト種別 | ツール | 目的 |
|-----------|-------|------|
| 型チェック | TypeScript (`tsc --noEmit`) | コンパイルエラーの検出 |
| ユニットテスト | Vitest | ユーティリティ関数・ヘルパーの検証 |
| E2E テスト | Playwright | ゴールデンパス（主要フロー）の検証 |
| 手動テスト | ブラウザ | Mapbox UI・アニメーション・Directions API の目視確認 |

---

## 2. ユニットテスト

### 2.1 対象

`lib/utils.ts` および `types/index.ts` のユーティリティ関数。

### 2.2 テストケース

```typescript
describe('utils', () => {
  it('cn() がクラス名を正しく結合する')
})

describe('types helpers', () => {
  it('getDayColor(1) が #3B82F6 を返す')
  it('getDayColor(8) が #EC4899 を返す')
  it('getDayColor(9) が getDayColor(1) と同じ値を返す（8色サイクル）')
  it('getCategoryIcon("spot") が 📍 を返す')
  it('getCategoryIcon("restaurant") が 🍽️ を返す')
  it('getCategoryIcon("hotel") が 🏨 を返す')
  it('getCategoryIcon("transport") が 🚗 を返す')
  it('getCategoryIcon("unknown") が 📍 を返す（フォールバック）')
})
```

## 5. テスト実行コマンド

```bash
# 型チェック
npm run type-check

# ユニットテスト
npm run test

# E2E テスト（ローカル）
npm run dev &
npm run test:e2e

# E2E テスト（UI モード）
npm run test:e2e:ui

# ビルド確認
npm run build
```
