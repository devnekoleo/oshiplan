# Viamaps ビジネス企画書

Google マイマップ × tabiori の機能を持つ旅行地図サービス

2026年5月 / **Version 3.0（フルリビルド版）**

---

## 0. v3.0 リビルドの背景

### 0.1 v2.0 までの課題

実装を進めた結果、以下の構造的な問題が判明した。

1. **操作フローの破綻**：地図上の地点を1つ選択するとモーダル/パネルが開いて、ポイントとして登録するまで他の操作（別地点を見る・パネルを閉じる・日程切替）が事実上できなくなる「操作ロック」状態が頻発。
2. **状態管理の暗黙性**：「pending pin」「編集中ポイント」「選択中ポイント」「ホバー中ポイント」が同じ UI 領域を奪い合い、ユーザーが「いま自分が何モードか」を把握できない。
3. **Mapbox の限界**：Places の評価情報・写真・営業時間など旅行プランニングに必須の情報が取りにくい。
4. **保存ライフサイクル不明瞭**：パネルを閉じるだけで作業が消える／silent fail する。

### 0.2 v3.0 の方針

- **地図プロバイダを Google Maps Platform に刷新**（Places/Directions/Geocoding 一体で利用）
- **操作モデルを「常にノンモーダル」に再設計**（地図クリックは常に有効、パネルは独立して開閉可能）
- **明示的なモード切替 UI**（閲覧モード／追加モード／編集モード／描画モードをトグルで明示）
- **下書き保存の明示化**（pending ポイントは画面上部に「未保存」バッジで常時可視化、明示破棄か保存のみで消える）
- **コア体験はそのまま継承**：日程管理 × 地図 × ← → ナビゲーション × 共有

## 1. エグゼクティブサマリー

Viamaps は **Google マイマップ**の地図機能と **tabiori** の旅程管理機能を統合した旅行地図 Web サービスである。

地図上にポイントを作り、**日程（Day 1・2・3…）ごとに整理**し、Google Directions API による**実際の道路ルート**を自動表示。「← →」ボタンで**地図を閉じることなくポイント間をスムーズにナビゲーション**できるコア体験に加え、チェックリスト・費用管理・描画ツール・共同編集を備える。

v3.0 では「**地点を触ってもUIが固まらない**」ことを最優先に再設計する。

### 1.1 解決する課題

| サービス | 課題 | Viamaps の解 |
|---------|------|-------------|
| Google マイマップ | ポイントAを見る→地図に戻る→繰り返しが苦痛。日程管理が弱い | 日程ベース + ← → ナビ |
| tabiori | 地図機能が限定的。道路ルートや距離が確認しにくい | Google Maps + Directions 統合 |
| 自社 v2.0 | 操作ロック・状態不明・保存喪失 | ノンモーダル UI + 下書きバッジ |

### 1.2 ターゲットユーザー

- **旅行計画者**：友人・家族との旅行を日程単位で地図にまとめたい（メイン）
- **旅行記録者**：行った場所・写真・感想・費用を地図に残したい
- **観光情報共有者**：おすすめスポットをルートつきでビジュアル共有したい
- **インバウンド事業者・宿泊施設**：周辺観光マップとして自社サイトに埋め込みたい

## 2. プロダクト概要

### 2.1 コアバリュー

> 「地図を何度も開き直さなくていい。日程ごとに整理して、← → だけで旅が進む。」

**v3.0 追加バリュー**：
> 「いつでも別の場所をクリックできる。何が下書きで、何が保存済みか、一目で分かる。」

### 2.2 競合サービスとの差別化

| 機能 | Google マイマップ | tabiori | Viamaps v3.0 |
|------|-----------------|---------|---------|
| ポイント間移動 | 毎回地図を開き直す | リスト中心 | **← → でアニメーション移動** |
| 日程管理 | 弱い（レイヤー） | Day 1・2・3… | **Day 1・2・3… + 地図連動** |
| 道路ルート | あり | なし | **Google Directions API** |
| 距離・所要時間 | あり | なし | **自動表示（日程ヘッダー）** |
| Places 情報統合 | あり | なし | **写真・評価・営業時間・電話を自動取得** |
| チェックリスト | なし | あり | あり（持ち物・やること） |
| 費用管理 | なし | あり | あり（ポイントごと + 日程集計） |
| 描画ツール | あり | なし | あり（ライン・ポリゴン） |
| 共同編集 | あり（Google アカウント） | あり | **招待リンクで誰でも編集可** |
| 埋め込みコード | あり | なし | あり（iframe + 軽量 API） |
| 共有 | Google ログイン要 | アプリ要 | **ログイン不要の閲覧専用リンク** |
| モバイル操作性 | 重い | 専用アプリ | スマホ最適化 Web (PWA) |

### 2.3 主要機能（v3.0）

#### 2.3.1 マップ作成・編集
- マップに名前・説明・カバー画像を付けて作成
- 地図クリックまたは **Google Places Autocomplete** でポイント追加
- ポイントに **Place ID** を紐付けて Google Places 詳細情報（写真・評価・営業時間・電話）を自動取得
- ポイントに独自情報（時刻・費用・カテゴリ・メモ・マーカー色）を追加
- **日程（Map Days）ごとにポイントを整理**
- ポイントのドラッグ並び替え／日程間ドラッグ移動
- **地図スタイル切替**（道路 / 衛星 / ハイブリッド / 地形）
- **描画ツール**（ライン・ポリゴン・色設定）

#### 2.3.2 日程管理（tabiori 相当）
- Day 1・Day 2・Day 3… を追加・並べ替え・削除
- 各日程に日付・タイトル・色・メモを設定
- 日程ごとに色分けされたマーカー・ルート線
- **Google Directions API** による実際の道路ルート + 距離・所要時間（車/徒歩/自転車/公共交通切替）
- 日程ごとの合計距離・合計費用・所要時間サマリー

#### 2.3.3 チェックリスト（tabiori 相当）
- 持ち物リスト（packing）
- やることリスト（todo）
- 担当者割り当て（共同編集者がいる場合）

#### 2.3.4 ビューアモード（コア差別化機能）
- デスクトップ：左サイドパネル（日程・ポイント一覧）+ 右地図
- 「←」「→」キーまたはボタンで次のポイントに **flyTo** アニメーション
- 日程フィルタ（Day 1 だけ表示など）
- 画像スライダー・Google Places の写真・口コミ要約

#### 2.3.5 共同編集
- **招待リンク**（閲覧／コメント／編集の3権限）
- リアルタイムカーソル表示（オプション）
- 編集者ごとのアクティビティログ
- マップ単位のロック（同時編集競合時の楽観ロック）

#### 2.3.6 共有・埋め込み
- 公開 ON で URL 共有（ログイン不要）
- **iframe 埋め込みコード**生成（高さ・幅・初期日程指定可）
- OGP 画像自動生成（マップサムネ + タイトル）
- 印刷用レイアウト（しおり PDF 出力）

### 2.4 UXフロー再設計（v3.0 の肝）

#### ❌ v2.0 までの問題フロー
```
[地図クリック] → 緑のpinとPointPanelが開く → 別地点クリック不可 → 保存しないと閉じられない
```

#### ✅ v3.0 の新フロー（ノンモーダル）
```
[地図クリック] → 下書きピン配置（画面上部に「未保存の地点があります」バッジ）
              ↓
              別地点クリック可（下書きピンは保持／移動）
              パネルは右側スライドオーバー（常時開閉可・閉じても下書きは残る）
              日程切替・スクロール・他ポイント編集すべて並行可能
              ↓
[「保存」or「破棄」] でのみ下書き解消
```

**モード明示トグル**（地図右上）：
- 👁 閲覧
- 📍 地点追加（クリックで下書き）
- ✏️ 描画
- 🗺 計測

各モードは色付きバッジで地図縁に常時表示。

### 3.2 コスト構造（Google Maps Platform）

**前提**：MAU 1,000 / アクティブユーザー1人あたり月間 Places 20回・Directions 10回・Geocoding 5回

| API | 月間呼出 | 単価 | 無料枠 | 月額コスト |
|-----|---------|------|-------|----------|
| Maps JavaScript API | 50,000 loads | 無制限無料 | - | ¥0 |
| Places Autocomplete | 20,000 | $2.83/1K | 10,000 | ≈ $28 ≈ ¥4,200 |
| Places Details | 10,000 | $17/1K | 10,000 | ¥0 |
| Directions API | 10,000 | $5/1K | 10,000 | ¥0 |
| Geocoding | 5,000 | $5/1K | 10,000 | ¥0 |
| **合計** | | | | **≈ ¥4,200/月** |

→ MAU 1,000 で ¥4,200 のインフラコスト。Plus プラン 10 件で回収可能。

**スケール時**：MAU 10,000 で約 ¥80,000/月。Starter プラン ($100/月) 切替も検討。

## 4. リスクと対応

| リスク | 影響 | 対応 |
|-------|------|------|
| Google Maps API 値上げ | コスト急増 | Mapbox フォールバック実装、SKU キャッシュ |
| Places 利用規約違反（情報の独自保存制限） | アカウント停止 | Place ID のみ保存、表示時に都度フェッチ |
| Supabase 障害 | サービス停止 | RPO 24h バックアップ、ステータスページ提供 |
| 著作権侵害ポイント | 通報リスク | 通報フォーム + 24h 内対応 |
| 共同編集の競合 | データ破損 | 楽観ロック + 自動マージ + 履歴復元 |

## 5. v3.0 マイグレーション戦略

### 5.1 既存ユーザー対応

v2.0 で作成済みのマップ（Mapbox 由来データ）に対して：
1. ポイントの lng/lat は座標としてそのまま継承
2. Google Geocoding で Place ID を逆引き取得（バックグラウンドジョブ）
3. Place ID 取得失敗の場合は「カスタム地点」として保持
4. ユーザーには「v3.0 にアップグレード完了」通知のみ
