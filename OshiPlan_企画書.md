# OshiPlan（オシプラン）ビジネス企画書

全ジャンル対応・推し活遠征プランナーアプリ

2026年5月 / Version 1.0

---

## 1. エグゼクティブサマリー

OshiPlan（オシプラン）は、アイドル・アーティスト・2.5次元・アニメ聖地巡礼など、あらゆるジャンルの「推し活遠征」を、AIが自動でワンストップ計画化するiOS/Androidネイティブアプリである。

公演情報やイベント名を入力すると、最寄り会場・交通手段・宿泊・物販列の並びタイミング・聖地巡礼スポットまでを含む遠征プランをAIが数秒で生成。サブスクリプション課金（月額480円）とアプリ内追加機能の組み合わせで、リリース18ヶ月以内に月次売上20万円超を目指す。

### 1.1 ターゲット

- **コア層**：月1回以上ライブ・公演・聖地巡礼に行く20〜30代の推し活実践者
- **サブ層**：地方在住で都市部遠征が必要なファン、初心者ファン
- **想定有料ユーザー**：1,000人（コア層の0.数%程度）

### 1.2 損益見通し（18ヶ月目）

有料ユーザー1,000人 × 月額480円 = 月次売上48万円。アプリストア手数料15〜30%、AI APIコスト等を差し引いた粗利は月25〜30万円を見込む。本業給与の補完として十分な水準。

## 2. 市場分析

### 2.1 市場規模

矢野経済研究所による2024年「オタク」市場の調査では、アイドル市場、アニメ市場を中心に推し活関連の市場が拡大している。コロナ後のライブ・現場復帰により、遠征需要は急回復している。

- 国内ライブ市場（コンサートプロモーターズ協会）：年間6,000億円規模、コロナ前を超え過去最高水準
- 推し活経済圏：関連グッズ・遠征費を含めると年間8,000億円超とする推計もある
- Z世代・ミレニアル世代の月平均推し活支出は20,000円超、うち遠征費は最大の支出項目

### 2.2 ユーザー課題（ペインポイント）

Twitter/X、TikTok、知恵袋などの定性調査から、推し活遠征の代表的な悩みは以下である：

1. 公演情報を見てから、交通・宿・物販列・聖地巡礼を別アプリで毎回調べるのが面倒
2. 初めての会場・地域だと、最適な動線が分からず時間と費用をロスする
3. 物販列の並び始め時間や、お見送り/出待ち動線が分からない
4. 複数公演の遠征計画を、同行者と共有・調整するのが煩雑
5. 遠征記録が散在し、後から振り返れない

### 2.3 競合分析

| カテゴリ | 代表サービス | カバー範囲 | OshiPlanとの差 |
|----------|-------------|-----------|---------------|
| 汎用旅行アプリ | Google Maps / Trip.com | 交通・宿のみ | 推し活文脈の理解なし |
| チケット管理 | チケットぴあアプリ等 | チケットのみ | プラン作成機能なし |
| 聖地巡礼アプリ | 舞台めぐり等 | アニメ・地図のみ | 遠征プラン機能なし |
| Excel/メモ | 手動管理 | 全部 | AIなし、手作業が大変 |

> **結論：推し活遠征を「ワンストップで自動計画する」アプリは2026年時点で日本市場に存在せず、明確な空白地帯がある。**

## 3. プロダクト概要

### 3.1 コアバリュー

> 「推し活の遠征計画にかかる3時間を、3分に短縮する」

### 3.2 主要機能

#### 3.2.1 AI遠征プラン自動生成（コア機能）

公演URL（チケットぴあ・イープラス等）または公演名を入力すると、以下を含むプランをAIが生成する：

- 出発地から会場までの最適交通手段（飛行機/新幹線/夜行バス）と所要時間・概算費用
- 会場周辺の宿泊候補（推し活民に人気のホテル、コインロッカーの位置）
- 物販列の推奨並び時間（過去のSNS情報をクロールしたAI推定）
- 開演前後の食事スポット（推しグッズ持ち込み可能な店、近場で時短）
- 聖地巡礼スポット候補（アニメジャンル時のみ）

#### 3.2.2 遠征しおり共有機能

生成したプランを「しおり」化し、同行者とリンクで共有。LINE/Twitterに投稿しやすいデザイン。

#### 3.2.3 遠征記録（公演アーカイブ）

過去の遠征プランを保存し、写真や感想を追加できる。年間遠征費・参戦数などを可視化。

#### 3.2.4 推しタイムライン

登録した推しの公演情報を自動収集し、通知。チケット先行・一般発売のリマインダー機能付き。

### 3.3 機能優先度（MVP範囲）

| 機能 | 優先度 | リリースフェーズ |
|------|--------|----------------|
| AI遠征プラン自動生成 | **最優先** | MVP（初回リリース） |
| プラン保存・編集 | **最優先** | MVP（初回リリース） |
| しおり共有（リンク共有） | 高 | MVP（初回リリース） |
| 推しタイムライン | 中 | v1.1（3ヶ月後） |
| 遠征記録・年間レポート | 中 | v1.2（6ヶ月後） |
| 同行者リアルタイム共同編集 | 低 | v2.0（12ヶ月後） |

## 5. マーケティング戦略

### 5.1 集客チャネル

広告費ゼロ運用を前提に、推し活民の情報収集経路を集中攻略する。

#### 5.1.1 X（旧Twitter）SNSマーケティング（主軸）

- 推し活民の情報源NO.1。「初現場」「初遠征」「○○ライブ」などのキーワードで困っている投稿に有益情報を返信して認知拡大
- 生成したプランのスクリーンショットを「これXXX公演用に作ったよ」と無料公開
- ハッシュタグ「#OshiPlan」「#推し活AI」での自然拡散を狙う

#### 5.1.2 TikTok・Instagram Reels

- 「神アプリ見つけた」系の発見コンテンツ。Z世代へのリーチに最適
- 実際の遠征Vlogの中でアプリを使うシーンを差し込む

#### 5.1.3 SEOコンテンツ

- 「○○ 遠征 持ち物」「△△会場 アクセス」など、推し活民の検索ロングテールを大量取得
- 会場ごとのガイド記事（東京ドーム、Kアリーナ、横アリ等）でドメイン強化

### 5.2 ローンチ戦略

1. **プレローンチ**：X上で「開発中アプリのβテスター募集」投稿でメール100件収集
2. **β版リリース**：100名にClosed招待。フィードバックでブラッシュアップ
3. **正式リリース**：App Store/Google Playで公開、Xでローンチ告知
4. **初週施策**：友達招待キャンペーン（紹介者・被紹介者の双方にPremium 1ヶ月無料）

### 5.3 リテンション施策

- **週次プッシュ通知**：登録した推しの公演情報の告知
- **公演前日リマインダー**：「明日は○○です、遠征プランの確認をどうぞ」
- **年間遠征レポート**：年末に「今年あなたは10公演、合計15万円の遠征をしました」的なまとめ。Spotify Wrappedのような共有シーン創出

## 6. リスクと対策

| リスク | 内容 | 対策 |
|--------|------|------|
| **公演情報の取得制限** | チケットぴあ等が公式APIを提供していない。スクレイピングは規約違反リスク | ユーザー入力ベースを基本とし、自動取得は公開SNSの公演ハッシュタグ等の合法範囲に限定 |
| **AI APIコスト** | 無料ユーザーの大量利用でAPI費用が売上を超過する可能性 | 無料枠を月3回に絞る。レート制限・キャッシュ活用でコスト最適化 |
| **競合大手の参入** | Yahoo!、楽天、JTBなどが類似機能をリリースする可能性 | 推し活コミュニティとの密接な関係構築。ニッチ機能の継続深化 |
| **個人開発者の継続性** | 本業多忙時に開発・サポートが滞るリスク | 障害自動通知の整備、解約は自動化、サポートはチャットボット中心 |
| **二次創作・肖像権** | ユーザー投稿で推しの写真等が無断使用される可能性 | 利用規約で個人利用に限定。報告窓口設置、迅速な削除運用 |
| **副業規定** | 本業の副業規定との適合 | 事前に就業規則確認・所定の申請手続を完了 |

## 8. 成功指標（KPI）

### 8.1 ノーススター指標

「月額有料ユーザー数」を最重要指標とする。

### 8.2 ダッシュボードで追う指標

- DAU / MAU比（プロダクトの粘着性）
- プラン生成数（コア機能の使用頻度）
- 無料→有料転換率（プロダクトの価値証明）
- Day7 / Day30 リテンション率
- チャーン率（月次）— 5%以下を目標
- CAC（顧客獲得コスト）— 広告費を使わない方針なので時間コスト換算
- LTV（顧客生涯価値）— 平均継続月数×単価

### 8.3 撤退・継続判断ライン

| 判断 | 条件 |
|------|------|
| 継続 | 6ヶ月で有料ユーザー300人到達 |
| ピボット検討 | 6ヶ月で有料ユーザー100人未満 |
| 撤退検討 | 12ヶ月で売上が運用コストを下回り続ける場合 |
